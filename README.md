# X5Tech Hiring App

Веб-приложение на Flask для автоматизированного первичного отбора кандидатов.  
Сервис поддерживает регистрацию HR и соискателей, заполнение анкеты, оценку профиля по баллам и просмотр заявок в профиле HR.

## Возможности

- Регистрация и вход для двух ролей: HR и соискатель.
- Заполнение анкеты кандидата.
- Автоматический скоринг заявки.
- Оценка по образованию, опыту, GitHub, портфолио и текстовому описанию.
- Разделение вакансий на массовые, профессиональные и стажировки.
- Просмотр заявок HR-ом в личном кабинете.
- Отображение статуса заявки в профиле соискателя.
- Хранение данных в SQLite.

## Стек технологий

- Python
- Flask
- SQLite
- HTML / CSS

## Как это работает

1. Соискатель проходит регистрацию или входит в систему.
2. Заполняет анкету с данными об образовании, опыте, GitHub и портфолио.
3. Приложение рассчитывает баллы по нескольким критериям.
4. Заявка сохраняется в базе данных.
5. HR видит список кандидатов, отсортированный по результатам скоринга.
6. Соискатель может отслеживать статус своей заявки.

## Основные роли

### HR
- Регистрируется по ключу доступа.
- Видит все заявки кандидатов.
- Смотрит ранжирование по баллам.
- Принимает решения по кандидатам.

### Соискатель
- Регистрируется в системе.
- Заполняет анкету.
- Получает статус заявки в личном кабинете.

## Правила скоринга

Система оценивает кандидата по следующим параметрам:

- Образование.
- Опыт работы.
- GitHub-проект.
- Наличие портфолио.
- Текстовое описание опыта и навыков.

Дополнительно учитывается возраст кандидата: если возраст меньше 18 или больше 50, заявка помечается как неподходящая.

## Конфигурация

По умолчанию база данных создаётся в файле `app.db` рядом с приложением.  
Порт можно задать через переменную окружения `PORT`.

Пример:

```bash
PORT=8000 python app.py
```

## Важные замечания

- Пароли хранятся в базе данных в открытом виде, поэтому для боевого проекта это нужно заменить на хеширование.
- Ключ доступа HR сейчас задан в коде и должен быть вынесен в переменные окружения.
- GitHub-оценка зависит от публичного API GitHub.
- Для полноценной работы нужны шаблоны в папке `templates` и стили в `styles`.
